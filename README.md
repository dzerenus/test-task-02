
# Приложение для учета личных финансов

Полное техническое задание описано в файле [Техническое Задание.txt](Техническое%20Задание.txt).  

Задание выполнено на `.NET Core 10`.

### Архитектура

При выполнении работы я уделил своё основное внимание архитектуре приложения и структурам данных.  
Базово оно разделено на четыре уровня.  

+ `Bank.Core` - содержит корневые иммутабельные типы данных, с которыми работают все основные уровни;
+ `Bank.Storage` - хранилище данных - в его качестве выступает `EntityFramework` с базой данных `PostgreSQL`. Проект содержит `Entity` для моделей `Bank.Core` и настройки таблиц, сделанные через `Fluent API`, а так же отдаёт на внешние уровни сервисы взаимодействия с хранилищем.
+ `Bank.App` - содержит основные сервисы работы программы. Предполагается, что это единственный уровень, который может работать напрямую с `Bank.Storage`, все остальные должны взаимодействовать с хранилищем только через сервисы этого уровня, которые не позволят сделать операций, нарушающих логику работы приложения.
+ `Bank.Cli` - непосредственная точка входа в приложение. Я написал консольное приложение так, чтобы оно было поддерживало `Dependency Injection` и использовало в основе своей сервисы. 

Я стараюсь использовать иммутабельные модели данных, поэтому корневые модели имеют три конструктора: создание нового объекта, копирование объекта с целью редактирования его копии и полное восстановление объекта по всем полям.

Я уделил внимание архитектуре, структурам данным и чистоте кода, но сознательно ослабил внимание за эффективностью работы кода, поэтому на огромных данных приложение может оказывать более высокую нагрузку на процессор и ОЗУ, чем могло бы, но на небольших данных это некритично.

### Запуск приложения

Для запуска необходимо указать корректную `StorageConnectionString` для вашей базы данных `PostgreSQL` в `appSettings.json` проекта `Bank.Cli` и выполнить его компиляцию, приложение должно сразу начать работу.
